📘 DIMENSIONALITY REDUCTION 
🔹 1. What is Dimensionality Reduction?

Dimensionality Reduction is the process of reducing the number of input features (dimensions) in a dataset while preserving as much important information as possible.

In machine learning, datasets often contain many features, but:

Some features are irrelevant

Some are highly correlated

Some add noise

Dimensionality reduction removes these unnecessary dimensions and represents data using fewer, more meaningful features.

🔹 2. Why Dimensionality Reduction is Needed?

Reduces computational cost

Improves model performance

Prevents overfitting

Removes noise

Helps in visualization

Solves curse of dimensionality

🔴 Curse of Dimensionality

As the number of features increases:

Data becomes sparse

Distance-based algorithms perform poorly

Models require more data to generalize

🔹 3. Feature Selection vs Feature Extraction
Feature Selection	Feature Extraction
Selects subset of original features	Creates new features
Keeps original meaning	New features have no direct meaning
Example: selecting top 5 columns	Example: PCA, LDA

👉 Dimensionality Reduction mainly uses Feature Extraction

🔹 4. TYPES OF DIMENSIONALITY REDUCTION

Dimensionality reduction techniques are classified as:

Linear Methods

Non-Linear (Manifold Learning) Methods

Randomized Methods

Deep Learning Based Methods

Supervised vs Unsupervised Methods

🔹 5. LINEAR DIMENSIONALITY REDUCTION METHODS

These methods assume linear relationships between features.

📌 5.1 PCA – Principal Component Analysis
🔹 Type:

Linear

Unsupervised

Feature Extraction

🔹 Idea:

PCA finds new axes (principal components) that:

Capture maximum variance

Are orthogonal (uncorrelated)

🔹 Working:

Standardize data

Compute covariance matrix

Find eigenvalues and eigenvectors

Select top k eigenvectors

Project data onto new space

🔹 Key Points:

First component captures maximum variance

Used for compression and noise reduction

Does NOT use class labels

🔹 Applications:

Image compression

Face recognition

Data visualization

📌 5.2 LDA – Linear Discriminant Analysis
🔹 Type:

Linear

Supervised

Feature Extraction + Classification

🔹 Idea:

LDA finds feature space that:

Maximizes between-class variance

Minimizes within-class variance

🔹 Working:

Compute class means

Compute scatter matrices

Solve eigenvalue problem

Project data onto discriminant axes

🔹 Key Points:

Uses class labels

Number of components ≤ (number of classes − 1)

Better class separation than PCA

🔹 Applications:

Face recognition

Medical diagnosis

Pattern recognition

📌 5.3 ICA – Independent Component Analysis
🔹 Type:

Linear

Unsupervised

🔹 Idea:

Find components that are statistically independent, not just uncorrelated.

🔹 Key Points:

Used in signal processing

Stronger condition than PCA

Separates mixed signals

🔹 Applications:

EEG analysis

Audio signal separation

🔹 6. NON-LINEAR DIMENSIONALITY REDUCTION (MANIFOLD LEARNING)

Used when data lies on a curved manifold.

📌 6.1 t-SNE
🔹 Type:

Non-linear

Unsupervised

Visualization

🔹 Idea:

Preserve local similarities in low-dimensional space.

🔹 Key Points:

Excellent for visualization

Poor for large datasets

Not suitable for prediction

🔹 Applications:

Data exploration

Cluster visualization

📌 6.2 Isomap
🔹 Type:

Non-linear

Unsupervised

🔹 Idea:

Preserves geodesic distances (distance along the manifold).

🔹 Steps:

Find neighbors

Build neighborhood graph

Compute shortest paths

Apply MDS

📌 6.3 LLE – Locally Linear Embedding
🔹 Type:

Non-linear

Unsupervised

🔹 Idea:

Each point can be reconstructed from its neighbors.

🔹 7. RANDOMIZED DIMENSIONALITY REDUCTION
📌 Random Projection
🔹 Type:

Linear

Unsupervised

🔹 Idea:

Project data onto random lower-dimensional space while preserving distances.

🔹 Key Points:

Very fast

Scalable

Approximate results

🔹 8. DEEP LEARNING BASED DIMENSIONALITY REDUCTION
📌 Autoencoders
🔹 Type:

Non-linear

Unsupervised

Neural Network based

🔹 Architecture:

Encoder → Bottleneck → Decoder

🔹 Idea:

Learn compressed representation by reconstructing input.

🔹 Applications:

Image compression

Anomaly detection

Feature learning

🔹 11. WHEN TO USE WHICH?

PCA → Large dataset, correlated features

LDA → Classification problems

t-SNE → Visualization

Isomap / LLE → Non-linear structure

Autoencoder → Images, deep learning tasks
